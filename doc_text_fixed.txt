5.1  XP Calculation Logic
XP_earned = base_xp
            × difficulty_multiplier
            × streak_multiplier
            × time_bonus_multiplier
            × quest_type_multiplier

# Base XP by difficulty:
base_xp = { easy: 50, medium: 100, hard: 200, boss: 500 }

# Difficulty multiplier (from adaptive engine):
difficulty_multiplier = difficulty_modifier  # 0.5 – 2.0

# Streak multiplier:
streak_multiplier = { days 1-6: 1.00, days 7-13: 1.25, days 14-29: 1.50, days 30+: 2.00 }

# Time bonus (complete under 80% of estimated time):
time_bonus_multiplier = 1.2 if actual_time &lt; 0.8 * estimated_time else 1.0

# Quest type multiplier:
quest_type_multiplier = { theory: 0.8, coding: 1.0, debug: 1.1, build: 1.3 }

5.2  Level Progression Formula
# XP required to reach level N:
xp_for_level(N) = 100 × (N ^ 1.8)

# Current level calculation:
current_level = floor( (total_xp / 100) ^ (1/1.8) )

# Progress to next level (for XP bar):
progress_pct = (total_xp - xp_for_level(N)) / (xp_for_level(N+1) - xp_for_level(N))

6.3  Hybrid Code Evaluation Model
  ★ NEW SECTION  
Pure AI evaluation of code submissions is unreliable, inconsistent, and gameable. Cognarc uses a hybrid model: test cases provide ground-truth correctness; AI provides contextual feedback. This is the only responsible approach.
Layer
Method
What It Validates
Weight in Decision
Primary
Test cases via sandboxed subprocess
Correctness — does the code produce the right output?
Pass/fail gate — must pass to earn full XP
Secondary
AI feedback (Groq/Mixtral)
Code quality — readability, efficiency, edge cases handled
Advisory only — does not block XP award
Fallback
Self-report + honor system
User confirms completion for theory/research quest types
Used when no objective test is possible

Why AI-Only Evaluation Is Unreliable
Problem
Consequence
LLMs hallucinate correctness for incorrect code
Users get XP for wrong submissions — completely undermines the skill system
AI feedback is inconsistent across calls
Same code gets different scores — users lose trust in the system
AI cannot distinguish runtime errors from logic errors
Incorrect code may appear &quot;reasonable&quot; to the model in context
Prompt injection via code submissions is a real attack vector
Malicious users could manipulate evaluator behavior

Sandboxing Implementation
# Evaluation pipeline — Python subprocess with resource limits
import subprocess, json, resource

def run_sandboxed(code: str, test_cases: list, timeout: int = 10)-&gt; dict:
    results = []
    for tc in test_cases:
        try:
            proc = subprocess.run(
                [&quot;python3&quot;, &quot;-c&quot;, f&quot;{code}\nprint({tc[&apos;call&apos;]})&quot;],
                capture_output=True, text=True,
                timeout=timeout,          # hard timeout: 10 seconds
                # No network, no file system write (container-level)
            )
            passed = proc.stdout.strip() == str(tc[&quot;expected&quot;])
            results.append({&quot;passed&quot;: passed, &quot;output&quot;: proc.stdout.strip()})
        except subprocess.TimeoutExpired:
            results.append({&quot;passed&quot;: False, &quot;output&quot;: &quot;TIMEOUT&quot;})
    pass_rate = sum(r[&quot;passed&quot;] for r in results) / len(results)
    return {&quot;pass_rate&quot;: pass_rate, &quot;results&quot;: results, &quot;passed&quot;: pass_rate &gt;= threshold}

13.3  Behavioral Adaptation vs Linear Learning
Linear learning assumes every user is identical. Cognarc&apos;s behavioral adaptation engine treats every user as a unique signal source. The same skill node generates different quests for two users at different difficulty modifiers, different streak states, and different completion rate histories.
